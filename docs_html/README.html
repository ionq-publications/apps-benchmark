qbenchmark - README

qbenchmark Documentation

Overview

qbenchmark is a quantum computing benchmarking framework that provides a standardized way to run and evaluate quantum algorithms across different quantum backends.

Features

  • Unified Interface: Abstract backend and algorithm interfaces for consistent benchmarking
  • Built-in Backends: Support for Mock, Qiskit Aer, and other quantum simulators
  • Built-in Benchmarks: Chemistry (VQE), Optimization (QAOA), and more categories
  • DIY Plugin System: Easy registration of custom backends and benchmarks
  • CLI Tool: Simple command-line interface for running benchmarks
  • Category Execution: Run entire benchmark suites with single commands

Installation

pip install -e .

Quick Start

List available benchmarks and backends

qbenchmark list
qbenchmark list --backends

Run a single benchmark

qbenchmark run --backend qiskit_aer_sim --case-uuid <uuid>

Run all benchmarks in a category

qbenchmark run --backend qiskit_aer_sim --category chemistry

Add custom backend

qbenchmark add --backend my_custom_backend

Add custom benchmark

qbenchmark add --benchmark my_algorithm --category optimization

Architecture

Core Components

  • AbstractBackend: Base class for all quantum backends
  • AbstractAlgoRunner: Base class for all benchmark algorithms
  • BenchmarkCase: Dataclass representing a benchmark problem instance
  • Registry: System for discovering and managing backends/benchmarks

Directory Structure

qbenchmark/
├── core/              # Core abstractions and registry
├── backends/          # Built-in backend implementations
├── benchmarks/        # Built-in benchmark categories
│   ├── chemistry/
│   ├── optimization/
│   └── ...
├── primitives/        # Data structures and utilities
└── cli.py            # Command-line interface

Creating Custom Backends

Create a Python file in ~/local_dev/backends/:

from qbenchmark.core.backend import AbstractBackend

class MyCustomBackend(AbstractBackend):
    def submit_job(self, circuit, **kwargs):
        # Your implementation
        pass

    def get_job_status(self, job_id):
        # Your implementation
        pass

    def get_results(self, job_id):
        # Your implementation
        pass

Register it:

qbenchmark add --backend my_custom_backend

Creating Custom Benchmarks

Create algorithm runner in ~/local_dev/benchmarks/my_category/algorithms/:

from qbenchmark.core.benchmark import AbstractAlgoRunner

class MyAlgorithmRunner(AbstractAlgoRunner):
    def run_benchmark(self, backend, problem_instance, **kwargs):
        # Your implementation
        return score, metadata, time_taken_s

Create problem instances in ~/local_dev/benchmarks/my_category/problem_instances/:

{
  "uuid": "unique-id",
  "category": "my_category",
  "benchmark_name": "my_algorithm",
  "problem_instance_data": {
    "param1": "value1"
  }
}

Register it:

qbenchmark add --benchmark my_algorithm --category my_category
